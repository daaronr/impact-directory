# CLAUDE.md

This file provides guidance to Claude Code when working with code in this repository.

## Project Overview

**Impact Products and Services Directory** — a searchable catalog of organizations and offerings
with meaningful social or environmental commitments. Every entry links to specific, sourced impact
claims with confidence scores, risk levels, and exact original wording.

## Architecture

Static site, identical pattern to the Library of Things project:

- **Frontend**: React 18 + Vite 5 + Tailwind v3
- **Routing**: React Router v6 (client-side)
- **Search**: Fuse.js (client-side fuzzy search)
- **Data layer**: Static JSON files — no database, no backend, no auth in v1
- **Deploy**: Netlify (static hosting)

## Common Commands

```bash
npm install          # Install dependencies
npm run dev          # Start dev server (http://localhost:5173)
npm run build        # Production build to dist/
npm run consolidate  # python scripts/consolidate.py — merge JSON into public/data/all_data.json
npm run validate     # python scripts/validate_data.py — validate data quality
```

## Data Layer

All data is static JSON, fetched client-side at runtime from `/data/all_data.json`.

### Directory layout

```
data/
  sources/sources.json         # Source directories (Profit for Good, GWWC, etc.)
  organizations/               # One or more JSON files; consolidate.py merges them
  offerings/                   # One or more JSON files
  claims/                      # One or more JSON files
public/
  data/all_data.json           # Consolidated file — GENERATED by consolidate.py, do not edit manually
```

### Schema

See `public/data/all_data.json` for the full structure. Key fields:

**Organization**: id, slug, name, description, website, org_type, country, source_ids, tags

**Offering**: id, slug, name, org_id, offering_type, exactness (exact|collection|inferred), category, tags

**Claim**: id, offering_id, org_id, claim_type, claim_text (exact original wording), parsed_value,
           confidence (0-1), risk_level (low|medium|high), source_id, beneficiary, cause_area,
           valid_from, valid_until

## Adding a New Organization

1. Add an entry to `data/organizations/organizations.json` (or create a new file in that directory)
2. Add any offerings to `data/offerings/offerings.json`
3. Add claims to `data/claims/claims.json` — use the exact original wording in `claim_text`
4. Run `python scripts/consolidate.py` to regenerate `public/data/all_data.json`
5. Run `python scripts/validate_data.py` to check for errors

## Adding a Source Importer

Add a new file in `scripts/importers/` extending `BaseImporter` from `base_importer.py`.
See `profit_for_good.py` and `gwwc.py` for stub examples with implementation notes.

## Key Design Decisions

1. **Static JSON, no database** — v1 is entirely static; all data in `public/data/all_data.json`
2. **Exact claim wording** — `claim_text` must be the original verbatim statement, never paraphrased
3. **Exactness levels** — exact / collection / inferred; see About page for definitions
4. **Confidence scores** — 0-1 float; High = 0.9+, Medium = 0.7+, Low = below 0.7
5. **No decorative emojis** — use plain text and Tailwind classes for visual hierarchy

## v2 Plan

A planned v2 will add community features (corrections, user submissions) via a lightweight SQLite
API on Linode. The static Vite + React frontend will remain; the API will be a thin layer on top.
No database service (no Postgres, no Supabase) is needed for v1.

## Deployment

Netlify static hosting. Required GitHub secrets:
- `NETLIFY_AUTH_TOKEN` — personal access token
- `NETLIFY_SITE_ID` — from Netlify dashboard
